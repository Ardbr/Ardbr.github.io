CMake 的使用

发表于2024-11-12|更新于2024-11-14
|浏览量:

常用命令

  1. 指定 cmake 的最低版本

  2. 指定项目名称

  3. set 命令的使用，例如指定 C++ 的标准、指定文件输出路径等

    • 指定 C++11 的标准

    • 指定制作的库的输出路径，一般放置在项目根目录的 lib 文件夹中

    • 指定可执行程序的输出路径，一般我们都放置在项目根目录的 bin 文件夹中

  4. 搜索文件

    • 方法 1: file 命令

      • 参数 1: GLOB 是 file 命令的一个参数，表示“全局匹配”，用于根据指定的模式查找文件。
      • 参数 2: 将查找到的文件保存在该变量中。
      • 参数 3: 匹配模式，示例中为匹配 CMAKE_CURRENT_SOURCE_DIR 宏指定的目录下的所有以 .cpp 结尾的文件。

    • 方法 2: 待补充，方法 1 就挺好用

  5. 制作一个库文件

    • 参数 1: 表示生成库的名字，注意这只是省略的写法，真正的库名为 libcalculate.lib 这种。
    • 参数 2: 表示生成库的类型, STATIC 表示生成静态库，SHARED 表示生成动态库（共享库）。
    • 参数 3: 表示用于生成库所需要的源文件。

  6. 打印信息

  7. 引用一个库文件：

    • 首先要包含头文件。因为头文件中有如何使用库文件中的接口，有了头文件才能指导我们里面有什么函数可以使用。
    • 然后应该告知要去哪里搜索我们的库文件，我们要指定一个具体的搜索目录。
    • 最后，我们便可以链接这个目录中的某个具体库使用了～

  8. 生成可执行文件

    参数 1: 生成可执行文件的名称。
    参数 2: 所需要用到的源文件。

常用的宏

宏名称 解释
CMAKE_CURRENT_SOURCE_DIR 当前 CMakeLists.txt 所在的目录
PROJECT_SOURCE_DIR 项目的根目录，注意在多 CMake 嵌套的工程中与 CMAKE_CURRENT_SOURCE_DIR 不一定等价
CMAKE_CXX_STANDARD CMake 要求的 C++ 最低版本
LIBRARY_OUTPUT_PATH 制作的库的输出（保存）路径
EXECUTABLE_OUTPUT_PATH 生成的二进制可执行程序的输出路径
