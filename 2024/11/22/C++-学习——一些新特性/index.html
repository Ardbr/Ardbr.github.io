std::optional<> 可选数据

  在一些情况下，我们需要读取一个文件，但我们怎么知道文件是否存在以及是否被成功读取呢？一般来说，如果成功读取，我们可以返回文件的字符串表示，而失败时，由于返回类型必须一致，我们可以返回一个空字符串 ""。但是这样就有一个问题，如果我们成功读取到了文件内容，但文件中恰恰没有任何内容，也就是我们成功读取了，但返回的内容仍然是 ""。这样我们又该如何分辨到底有没有读取成功呢？   一种解决思路是再引入一个额外的布尔变量 success_flag 专门用于标明是否读取成功。但这样会有额外的开销。能否只用一个变量就能做到呢。这就是 C++17 引入的新特性可选 std::optional<> 可选数据类型。   我们只需要把返回值 string 设置为可选的，读取成功时给它赋值，而失败时留空，不做任何操作。那么我们就可以通过它里面是否有值来判断文件是否读取成功啦！

  std::optional<> 有成员函数 has_value() 返回布尔值表示是否有值，但类本身也是有布尔运算符的，所以可以直接 if (data) 来进行判断。   value_or() 函数非常有用，作用如其函数名所述，如果本身有值，那么就用本身的值，否则，就用传入的参数作为值，起到了一种 “默认” 值的作用。

与 Union 比较

  两者似乎都是在单一变量中能够存储多个值，但两者有很大的不同，例如在存储方式上。对于 Union 来说，不同的变量占据的是同一块内存，因此 Union 的总大小取决于所包含的所有类型中最大的那一个。而对于 std::variant，它更像是为每一个类型单独分配一个变量来接收，例如在内部类似 std::string data1, int data2 这种。因此它的大小是所有类型的总和。更正：经过后续测试发现，std::variant 的大小并不是各个类型大小之和，而是最大类型的占用大小，再加上额外的 size_t index 的 8 字节大小。似乎不同的平台结果会不同。。。 > Currently class variant has data members std::size_t type_index and data_type data. Here sizeof(data_type) will be the size of the largest type the variant includes. But together they will be at least sizeof(size_t) == 8 larger.

使用 g++ 编译器，在 macos M3 arm64 架构下，sizeof(std::string) == 24，因为 char * s; size_t capacity; size_t length; 总共需要 3 x 8 = 24 字节。

  总体来说，Union 占用的内存大小更少，但 std::variant 在技术上访问起来更加安全。
